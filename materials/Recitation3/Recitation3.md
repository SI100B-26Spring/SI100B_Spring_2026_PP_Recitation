# 零、函数作用域与函数对象（上上周内容）

<!--v-->

## 1. 调用函数时发生了什么？

- 每调用一次函数，Python 都会新建一个 **环境（environment）/作用域（scope）**
- 这个新作用域就像一个临时的小工作区
- 函数会：
  - 接收输入值
  - 把输入值绑定给形参
  - 执行函数体中的代码
  - 返回结果
- 函数返回后，这个局部作用域就会消失

<!--v-->

## 1. 调用函数时发生了什么？

```python
def f(x):
    x = x + 1
    return x

a = 3
b = f(a)
```

- 这里全局变量 `a` 仍然是 `3`
- 函数内部操作的是它自己的局部变量 `x`

<!--v-->

## 2. 全局作用域与局部作用域

- **全局作用域**：程序开始执行的地方
- **局部作用域**：调用函数时创建的作用域
- 同一个名字在不同作用域中可以表示不同的值

```python
def f(x):
    x = x + 1
    print("函数内部 x =", x)
    return x

x = 3
z = f(x)
print("函数外部 x =", x)
print("z =", z)
```

<!--v-->

## 2. 全局作用域与局部作用域

输出为：

```python
函数内部 x = 4
函数外部 x = 3
z = 4
```

- 函数里的 `x` 和外面的 `x` 不是同一个变量
- 因此函数内部修改 `x`，不会影响外部的 `x`

<!--v-->

## 3. 变量作用域的一个重要规则

- 在函数内部，**可以读取**外部定义的变量
- 但通常**不能直接修改**外部变量
- 在函数里给某个名字赋值时，Python 往往会把它当作局部变量

```python
def g(y):
    print(x)
    print(x + 1)

x = 5
g(x)
print(x)
```

- 这段代码可以正常运行
- 因为函数 `g` 只是读取了外部的 `x`

<!--v-->

## 3. 变量作用域的一个重要规则

```python
def h(y):
    x += 1   # 报错

x = 5
h(x)
```

- 这段代码会报错
- 因为 Python 认为 `x` 是函数内部的局部变量
- 但在使用它之前，并没有先给它赋值

<!--v-->

## 4. LEGB 规则

当 Python 遇到一个变量名时，会按下面顺序查找：

- **L = Local（局部）**
- **E = Enclosing（外层嵌套函数）**
- **G = Global（全局）**
- **B = Built-in（内置）**

一些常见的内置名字：

```python
len("abc")
abs(-7)
range(5)
```

<!--v-->

## 4. LEGB 规则

所以读代码时要问自己：

- 现在执行的是哪一行表达式？
- 当前在哪个作用域里？
- Python 会先去哪里找这个变量名？

<!--v-->

## 5. 函数也是对象

在 Python 中，函数也是 **对象（object）**

这意味着：

- 函数可以赋值给变量
- 函数可以作为参数传给另一个函数
- 函数也可以作为返回值

<!--v-->

## 5. 函数也是对象

```python
def is_even(i):
    return i % 2 == 0

my_func = is_even

a = is_even(3)
b = my_func(4)

print(a)   # False
print(b)   # True
```

注意：

- `my_func = is_even` 并没有调用函数
- 它只是让 `my_func` 和 `is_even` 指向同一个函数对象

<!--v-->

## 6. 函数作为参数

一个函数可以把另一个函数当作输入参数

```python
def calc(op, x, y):
    return op(x, y)

def add(a, b):
    return a + b

print(calc(add, 2, 3))
```

执行过程：

- 调用 `calc(add, 2, 3)`
- 在 `calc` 中，`op` 绑定到函数 `add`
- 然后执行 `op(x, y)`
- 也就是执行 `add(2, 3)`
- 最后结果为 `5`

<!--v-->

## 6. 函数作为参数

这种写法的好处是：

- 把“做什么运算”
- 和“整体程序怎么组织”
- 分开处理

<!--v-->

## 7. lambda 匿名函数

有时我们只需要一个很简单、只用一次的小函数

这时可以使用 `lambda`

```python
lambda x: x % 2 == 0
```

它表示：

- 参数是 `x`
- 返回值是表达式 `x % 2 == 0`

<!--v-->

## 7. lambda 匿名函数

例如：

```python
def apply(criteria, n):
    count = 0
    for i in range(n + 1):
        if criteria(i):
            count += 1
    return count

print(apply(lambda x: x % 2 == 0, 10))
```

- 这段代码统计从 `0` 到 `10` 中有多少个偶数
- `lambda` 很适合写这种简单、一次性的函数

<!--v-->

## 8. 例子：把函数作用两次

```python
def do_twice(n, fn):
    return fn(fn(n))
print(do_twice(3, lambda x: x**2))
```

执行过程：

先算一次：$fn(3) = 3^2 = 9$

再算一次：$fn(9) = 9^2 = 81$

所以输出结果是：

```python
81
```

<!--v-->

## 这节的核心结论

- 每次函数调用都会创建自己的作用域
- 函数可以像普通值一样传递和使用
- `lambda` 可以快速创建简单函数
- 把程序拆成多个函数，代码会更清晰、更容易组织

# 一、Sorting

<!--v-->

## 什么是排序

给定一个序列，把它按某种顺序重新排列。

通常默认是**非降序**（nondecreasing order）：

- 允许重复值
- 前面的元素不大于后面的元素

例如：

```text
Input:  [4, 1, 8, 5, 4]
Output: [1, 4, 4, 5, 8]
```

<!--v-->

## 排序对象不一定是数字

只要一个类型有明确的比较规则，就可以排序。

- 整数
- 浮点数
- 字符串
- 任何具有 total ordering 的对象

所以排序本质上是：

> 根据“比较”来重新组织数据。

<!--v-->

## 排序为什么重要

排序本身是基础操作，也常常作为别的任务的前处理步骤。

常见用途：

- 更快查找
- 去重
- 找中位数
- 统计频率
- 求交集
- 解决 2SUM 等问题

<!--v-->

## 应用 1：Binary Search

如果数据是无序的：

- 找一个元素通常比较麻烦
- 往往只能线性扫描

如果数据已经有序：

- 就可以做 **binary search**
- 每次看中间，排除一半区间

这就是“排序换查找效率”的典型例子。

<!--v-->

## 应用 2：去重

例如：

```text
Input:  [4, 2, 5, 1, 2, 4, 3]
Sort:   [1, 2, 2, 3, 4, 4, 5]
Output: [1, 2, 3, 4, 5]
```

核心思想：

- 先排序
- 再扫描一遍
- 删除相邻重复值

<!--v-->

## 应用 3：中位数 / 频率统计

- 找中位数：先排序后取中间元素
- 找出现次数最多的数：排序后，相同元素会聚到一起，便于统计

例如：

```text
S = [4, 2, 3, 2, 1, 3, 2]
Sort -> [1, 2, 2, 2, 3, 3, 4]
Most frequent = 2
```

<!--v-->

## 应用 4：集合交与 2SUM

### Set Intersection

两个**有序**数组求交集：

- 两个指针同时向前走
- 总时间为 $O(m+n)$

### 2SUM

若数组有序，要找两数之和为 $t$：

- 左右双指针夹逼
- 总时间为 $O(n)$

# 二、插入排序（Insertion Sort）

<!--v-->

## Insertion Sort 的核心想法

把数组分成两部分：

- 左边：已经排好序
- 右边：还没处理

每次从右边取一个元素，插入到左边的正确位置。

这就像整理扑克牌：

- 手里的牌始终保持有序
- 新摸到一张，就插进合适的位置

<!--v-->

## 算法过程

假设前 $k$ 个元素已经排好序。

接下来处理第 $k+1$ 个元素 $v$：

- 从后往前比较
- 找到第一个比 $v$ 小或等于的位置
- 把 $v$ 插进去
- 中间较大的元素整体右移

<!--v-->

## 示例

```text
S = [1, 3, 4, 6, 2, 5, 10, 7, 9]
```

前四个元素已经有序：

```text
[1, 3, 4, 6]
```

把 `2` 插进去后：

```text
[1, 2, 3, 4, 6, 5, 10, 7, 9]
```

然后继续插入 `5`, `10`, `7`, `9`。

<!--v-->

## 伪代码

```python
for i in range(1, len(A)):
    j = i
    while j > 0 and A[j - 1] > A[j]:
        A[j], A[j - 1] = A[j - 1], A[j]
        j -= 1
```

理解重点：

- 外层循环：依次处理每个新元素
- 内层循环：把当前元素往左移动到正确位置

<!--v-->

## 时间复杂度

最坏情况发生在：

- 数组本来是完全逆序的
- 每次新元素都要一路移动到最前面

第 $i$ 轮最多做 $i$ 次交换，因此总操作数最多为：

$$
1 + 2 + 3 + \cdots + (n-1) = \frac{n(n-1)}{2}
$$

所以：

$$
T(n)=O(n^2)
$$

<!--v-->

## Insertion Sort 的优缺点

### 优点

- 代码简单
- 额外空间很小，基本是 $O(1)$
- 当输入**几乎有序**时很快
- 适合很小规模的数据

### 缺点

- 最坏复杂度是 $O(n^2)$
- 对大规模数据不够高效

<!--v-->

## 为什么说它是 adaptive

如果数组已经接近有序，Insertion Sort 会很快。

例如：

```text
[1, 3, 2, 4, 6, 5, 7, 9, 10, 8]
```

每个元素离最终位置都不远，移动次数就很少。

所以它不是“总是慢”，而是：

> **对小规模、近似有序的数据特别实用。**

<!--s-->

# 三、递归（Recursion）

<!--v-->

## 递归的基本思想

递归就是：

> 用更小规模的同类问题，去解决当前问题。

递归通常包含两部分：

- **Base case**：最小情况，直接求解
- **Recursive step**：把大问题化成更小的问题

<!--v-->

## 迭代（Iterative） vs 递归（Recursive）

### Iterative

- 用 `for` / `while`
- 关注变量如何变化
- 关注循环什么时候停

### Recursive

- 关注问题如何缩小
- 关注 base case 是什么
- 关注函数如何调用自己

两者经常能解决同一个问题，只是写法不同。

<!--v-->

## 例子：乘法

可以把 $a \times b$ 理解成重复加法：

$$
a \times b = a + a \times (b-1)
$$

base case：

$$
a \times 1 = a
$$

对应代码：

```python
def mult(a, b):
    if b == 1:
        return a
    return a + mult(a, b - 1)
```

<!--v-->

## 常见递归例子

### 阶乘

$$
n! = n \cdot (n-1)!, \quad 1! = 1
$$

```python
def fact(n):
    if n == 1:
        return 1
    return n * fact(n - 1)
```

### 幂

$$
n^p = n \cdot n^{p-1}, \quad n^0 = 1
$$

<!--v-->

## 另一个例子：反转字符串

```python
def rev(s):
    if len(s) == 1:
        return s
    return rev(s[1:]) + s[0]
```

思想是：

- 先反转去掉首字符后的部分
- 再把首字符接到最后

这很符合“先解决更小问题”的思路。

<!--v-->

## 递归调用时发生了什么

每一次函数调用，都会创建一个**独立的作用域**。

例如 `fact(4)` 实际上会产生：

- `fact(4)` 等待 `fact(3)` 的结果
- `fact(3)` 等待 `fact(2)` 的结果
- `fact(2)` 等待 `fact(1)` 的结果
- `fact(1)` 返回 1
- 再一层层返回

所以递归调用是靠 **call stack** 管理的。

<!--v-->

## 栈（Stack） 的直观理解

- 新调用压入栈顶（push）
- 调用完成后从栈顶弹出（pop）

因此：

- 递归层数太深会爆栈
- Python 也有递归深度限制

所以递归虽然写起来简洁，但不是无限可用的。

<!--v-->

## 什么时候适合递归

适合递归的典型场景：

- divide and conquer
- 树结构
- 图遍历
- 回溯
- 问题天然能分成更小同类子问题

不一定适合递归的情况：

- 递归层数很深
- 迭代写法更直接
- 对性能特别敏感

<!--v-->

## 递归与迭代的取舍

### 递归优点

- 写法自然
- 思路清晰
- 对很多算法表达力强

### 递归缺点

- 有函数调用开销
- 占用额外栈空间
- 太深会触发 stack limit

所以实践中常见情况是：

> 递归更好理解，迭代往往更省资源。

# 四、如何分析程序复杂度

<!--v-->

## 为什么不能只看“运行时间”

直接测程序跑了多久，并不是统一标准，因为它受很多因素影响：

- 硬件
- 操作系统
- 编译器 / 解释器
- 输入数据

所以我们更关心：

> **随着输入规模变大，算法增长得有多快。**

<!--v-->

## 时间复杂度的三个核心问题

### 问题 1：输入不同，步数不同

解决：把复杂度写成输入规模 $n$ 的函数。

### 问题 2：同样大小的输入，运行时间也可能不同

解决：通常先分析 **worst case**。

### 问题 3：什么算“一步”

解决：用统一、粗粒度的模型，只抓主要增长项。

<!--v-->

## for 循环怎么分析

如果循环体每次执行算 1 步：

```python
for i in range(n):
    # do stuff
```

复杂度是：

$$
O(n)
$$

如果步长是 2：

```python
for i in range(0, n, 2):
    # do stuff
```

迭代次数约为 $n/2$，仍然是：

$$
O(n)
$$

<!--v-->

## 嵌套循环怎么分析

```python
for i in range(n):
    for j in range(2 * n):
        # do stuff
```

总步数：

$$
n \cdot 2n = 2n^2
$$

所以复杂度是：

$$
O(n^2)
$$

常数 2 忽略不计。

<!--v-->

## 变长内层循环

```python
for i in range(n):
    for j in range(i):
        # do stuff
```

总步数是：

$$
0 + 1 + 2 + \cdots + (n-1) = \frac{n(n-1)}{2}
$$

因此：

$$
O(n^2)
$$

<!--v-->

## while 循环怎么分析

### 例 1

```python
i = 0
while i < n:
    i += 1
```

运行 $n$ 次，所以是：

$$
O(n)
$$

<!--v-->

## while 循环怎么分析

### 例 2

```python
i = 1
while i < n:
    i *= 2
```

`i` 的取值是：$1, 2, 4, 8, \dots$

所以循环次数约为：$\lceil \log_2 n \rceil$

复杂度为：

$$
O(\log n)
$$

<!--v-->

## if-else 怎么分析

在 worst case 分析中：

- 不知道实际走哪个分支
- 就看**最慢的分支**

例如：

- 分支 A：$O(n)$
- 分支 B：$O(n^2)$

整体复杂度取较大者：

$$
O(n^2)
$$

<!--s-->

# 五、递归复杂度：Recurrence

<!--v-->

## 分析递归算法的两步

分析递归算法通常分两步：

1. 写出递推式（recurrence relation）
2. 解这个递推式

记 $S(n)$ 为输入规模为 $n$ 时的步数。

<!--v-->

## 例 1：简单递归

```python
def sum_n(n):
    if n == 1:
        return 1
    return n + sum_n(n - 1)
```

对应递推式：

$$
S(1)=1
$$
$$
S(n)=1+S(n-1)
$$

因为每一步只做常数工作，再递归到 $n-1$。

解得：

$$
S(n)=O(n)
$$

<!--v-->

## 例 2：每次缩小一半

```python
def bar(n):
    if n <= 1:
        return 0
    return 1 + bar(n // 2)
```

递推式：

$$
S(1)=1, \quad S(n)=1+S(n/2)
$$

不断展开：

$$
S(n)=1+1+1+\cdots
$$

直到规模变成 1，大约需要 $\log_2 n$ 次。

所以：

$$
S(n)=O(\log n)
$$

<!--v-->

## 例 3：分成两个子问题

```python
baz(n):
    baz(n/2)
    baz(n/2)
    # do n work
```

递推式：

$$
S(n)=2S(n/2)+n
$$

这是分析归并排序（Mergesort）的核心模板。

<!--v-->

## 递归树法（recursion tree）

对于

$$
S(n)=2S(n/2)+n
$$

每一层的总工作量都是 $n$：

- 第一层：$n$
- 第二层：$2 \cdot n/2 = n$
- 第三层：$4 \cdot n/4 = n$
- ……

树高约为 $\log_2 n$。

所以总工作量：

$$
S(n)=O(n\log n)
$$

<!--v-->

## 常见递推式记忆

- $S(n)=S(n-1)+1$ 对应 $O(n)$
- $S(n)=S(n/2)+1$ 对应 $O(\log n)$
- $S(n)=2S(n/2)+n$ 对应 $O(n\log n)$
- $S(n)=S(n-1)+n$ 对应 $O(n^2)$

这几类是初学阶段最常见的复杂度模型。

<!--s-->

# 六、归并排序（Mergesort）

<!--v-->

## Mergesort 的思想

Mergesort 是典型的 **divide and conquer**：

1. 把数组分成两半
2. 递归排序左半边
3. 递归排序右半边
4. 把两个有序数组 merge 成一个有序数组

base case：

- 数组长度为 1 时天然有序

<!--v-->

## Merge 这一步最关键

假设 `L` 和 `R` 都已经有序。

我们要把它们合并成新的有序数组 `A`。

做法：

- 维护两个指针 `i`, `j`
- 比较 `L[i]` 和 `R[j]`
- 较小者放入结果数组
- 对应指针右移
- 直到一边用完，再把另一边剩余元素接上

<!--v-->

## Merge 的伪代码

```python
def merge(L, R):
    A = []
    i = 0
    j = 0

    while i < len(L) and j < len(R):
        if L[i] <= R[j]:
            A.append(L[i])
            i += 1
        else:
            A.append(R[j])
            j += 1

    A.extend(L[i:])
    A.extend(R[j:])
    return A
```

<!--v-->

## Mergesort 的递推式

设 $S(n)$ 是排序 $n$ 个数的时间。

则：

$$
S(1)=1
$$
$$
S(n)=2S(n/2)+M(n)
$$

其中 $M(n)$ 是 merge 的复杂度。

由于 merge 只需要线性扫描一遍：

$$
M(n)=O(n)
$$

所以：

$$
S(n)=2S(n/2)+O(n)
$$

<!--v-->

## Mergesort 的时间复杂度

由递归树可得：

- 每层总代价是 $O(n)$
- 层数是 $O(\log n)$

因此：

$$
S(n)=O(n\log n)
$$

这比 Insertion Sort 的 $O(n^2)$ 在大规模输入下快得多。

<!--v-->

## Mergesort 的缺点

Mergesort 的主要问题不是时间，而是**空间**。

因为 merge 时通常需要额外数组：

- 左半部分临时存储
- 右半部分临时存储

因此额外空间不是 $O(1)$，而是线性级别。

所以它的特点是：

- **时间好**：$O(n\log n)$
- **空间占用较多**

<!--v-->

## Insertion Sort vs Mergesort

### Insertion Sort

- 时间：$O(n^2)$
- 空间：$O(1)$
- 小规模、近似有序时很不错

### Mergesort

- 时间：$O(n\log n)$
- 空间：需要额外辅助数组
- 大规模数据时更有优势

实际中常见做法：

> 小区间用 insertion sort，大区间用 mergesort。

<!--s-->

# 七、快速排序（Quicksort）

<!--v-->

## Quicksort 的核心思想

Quicksort 也是 divide and conquer。

步骤：

1. 选一个 **pivot**
2. 把数组分成两部分：
   - 小于等于 pivot 的放左边
   - 大于 pivot 的放右边
3. 对左右两边递归排序

如果 partition 做完，pivot 的最终位置就固定了。

<!--v-->

## Partition 的不变量

常见实现中，选最后一个元素作 pivot。

用两个指针 `i, j` 扫描时保持：

- `A[p..i] <= pivot`
- `A[i+1..j] > pivot`
- `A[j+1..r-1]` 还没处理

每看到一个 `<= pivot` 的元素，就把它交换到前面来。

<!--v-->

## Partition 伪代码

```python
def partition(A, p, r):
    x = A[r]
    i = p - 1
    for j in range(p, r):
        if A[j] <= x:
            i += 1
            A[i], A[j] = A[j], A[i]
    A[i + 1], A[r] = A[r], A[i + 1]
    return i + 1
```

这个 partition 是 **in-place** 的。

这也是 quicksort 的重要优势之一。

<!--v-->

## Quicksort 的递推式

设 partition 后两边规模分别为 $u$ 和 $v$，满足：

$$
u+v=n-1
$$

那么：

$$
S(n)=S(u)+S(v)+cn
$$

其中：

- $S(u)$：左边递归，$S(v)$：右边递归
- $cn$：partition 的线性代价

Quicksort 的关键就在于：

> **pivot 选得平不平衡。**

<!--v-->

## Best Case

如果每次都几乎均分：

$$
u \approx v \approx n/2
$$

则递推式近似为：

$$
S(n)=2S(n/2)+cn
$$

因此：

$$
S(n)=O(n\log n)
$$

和 mergesort 一样快。

<!--v-->

## Worst Case

如果每次都极端不平衡：

- 一边是 $n-1$
- 另一边是 0

则：

$$
S(n)=S(n-1)+cn
$$

解得：

$$
S(n)=O(n^2)
$$

这和 insertion sort 的最坏复杂度一样。

<!--v-->

## Average Case 为什么快

随机输入下，pivot 通常不会总是最小或最大。

即使分得不是完全均衡，比如：

- 一边是 $0.9n$
- 一边是 $0.1n$

递归树高度仍然是 $O(\log n)$。

而每层总 partition 工作量仍然是 $O(n)$。

所以平均情况下：

$$
S(n)=O(n\log n)
$$

<!--v-->

## Quicksort vs Mergesort

### Quicksort

- 平均情况快
- 原地 partition，额外空间少
- cache locality 更好
- 最坏情况可能退化到 $O(n^2)$

### Mergesort

- 时间更稳定，总是 $O(n\log n)$
- 需要额外空间

所以：

- 理论最坏情况：Mergesort 更稳
- 实际很多场景：Quicksort 很常用、很快

<!--s-->

# 八、大O（Big O）表示法

<!--v-->

## 为什么需要 Big O

Big O 用来比较函数增长速度，而不是比较精确值。

在算法里，这意味着：

- 不纠结常数
- 不纠结小规模输入
- 只关心输入很大时谁增长更快

<!--v-->

## Big O 的两个核心思想

### 1. 看渐近行为（asymptotically）

只看 $n$ 很大时的趋势。

### 2. 忽略常数因子

例如：

- $n$
- $2n$
- $100n$
- $0.01n$

它们都认为是同一个量级：

$$
O(n)
$$

<!--v-->

## 常见记号

- $f(n)=O(g(n))$：$f$ 增长不比 $g$ 快太多
- $f(n)=\Omega(g(n))$：$f$ 至少和 $g$ 一样大
- $f(n)=\Theta(g(n))$：$f$ 与 $g$ 同阶
- $f(n)=o(g(n))$：$f$ 严格低于 $g$
- $f(n)=\omega(g(n))$：$f$ 严格高于 $g$

在本课中，最常用的是：

- $O$
- $\Omega$
- $\Theta$

<!--v-->

## 用极限来理解

可以用比值极限直观判断：

- 若
  $$
  \lim_{n\to\infty} \frac{f(n)}{g(n)} < \infty
  $$
  则 $f(n)=O(g(n))$

- 若
  $$
  \lim_{n\to\infty} \frac{f(n)}{g(n)} > 0
  $$
  则 $f(n)=\Omega(g(n))$

- 若极限是一个正常的正有限常数，则
  $$
  f(n)=\Theta(g(n))
  $$

<!--v-->

## 一些基础例子

### 例 1

$$
n = O(2n)
$$

因为$\frac{n}{2n}=\frac12$

### 例 2

$$
10n^2 = O(n^2)
$$

因为常数 10 不影响阶。

### 例 3

$$
n = O(n^2)
$$

因为线性增长慢于二次增长。

<!--v-->

## 再看几个例子

### 正确

$$
n^2 = \Omega(n)
$$

$$
n = \Omega(\log n)
$$

### 错误

$$
n \neq O(\sqrt{n})
$$

$$
n^2 \neq \Theta(n)
$$

$$
n^2 \neq \Omega(2^n)
$$

因为指数函数最终增长远快于多项式。

<!--v-->

## 常见增长速度排序

从慢到快大致为：

$$
O(1)
< O(\log n)
< O(n)
< O(n\log n)
< O(n^2)
< O(n^3)
< O(2^n)
$$

记住这条链非常重要。

<!--v-->

## Big O 的性质

### 传递性

如果

$$
f(n)=O(g(n)), \quad g(n)=O(h(n))
$$

则

$$
f(n)=O(h(n))
$$

<!--v-->

## Big O 的性质

### 对称性

$\Theta$ 是对称的：

$$
f(n)=\Theta(g(n)) \iff g(n)=\Theta(f(n))
$$

<!--v-->

## Big O 的性质

### 常数属于 $O(1)$

任何常数时间操作都记作：

$$
O(1)
$$

<!--s-->

# 九、如何用 Big O 比较算法

<!--v-->

## 例子

假设两个算法的递推式分别是：

$$
S_1(n)=S_1(n-1)+n
$$

$$
S_2(n)=2S_2(n/2)+n
$$

可得：

$$
S_1(n)=\Theta(n^2)
$$

$$
S_2(n)=\Theta(n\log n)
$$

因为：

$$
n^2 = \Omega(n\log n)
$$

所以当 $n$ 足够大时，算法 2 更快。

<!--v-->

## 但 Big O 不是全部

Big O 告诉我们增长趋势，但不表示一切。

实际性能还受这些影响：

- 常数因子
- 内存访问模式
- cache locality
- 递归开销
- 实际输入分布
- 实现细节

所以结论通常是：

> **Big O 很重要，但不是唯一标准。**
